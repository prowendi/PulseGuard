{{/* ===== PUSH ===== */}}

Push 接入

把外部告警送入推送管线。Push token 来自 Channel rotate-token。

{{ template "apidoc-row" (mkAPI "POST" "/api/v1/push/{token}" "投递告警；用 push_token 鉴权（无需 session）" `{"title":"CPU high","host":"db01","level":"critical","_fingerprint":"db01-cpu","_buttons":[{"text":"我处理","callback":"ack:db01-cpu"}]}` `{"push_id":12345,"status":"queued"}`) }}

特殊字段（下划线前缀）：

  • _fingerprint: 同指纹第二次 push 走 editMessageText 合并到首条消息（去噪）
  • _template_id: 显式指定 Channel 绑定的 template_id（覆盖 default + 条件路由）
  • _buttons: inline keyboard，callback 形如 ack:<fingerprint> 用户点击即写 alert_acks
  • dedup_key: 启用 Channel dedup_window_s 时的指纹
  • 其余字段任意，模板里用 .field | escMD 风格的 Go template action 取（具体看模板编辑器示例）
{{/* ===== BOTS ===== */}}

Bots

租户私有推送 bot；bot_token AES-GCM 加密入库，列表只回显末 4 位。支持 Telegram + Lark/飞书，按 platform 字段路由。

{{ template "apidoc-row" (mkAPI "GET" "/api/v1/bots" "列出本租户全部 bot" "" `{"items":[{"id":1,"name":"prod","platform":"telegram","bot_token_last4":"Y5EQ","enabled":true}]}`) }} {{ template "apidoc-row" (mkAPI "POST" "/api/v1/bots" "新建 bot；创建后自动启动长轮询" `{"name":"prod","platform":"telegram","bot_token":"123456:ABC","description":""}` `{"id":1,...}`) }} {{ template "apidoc-row" (mkAPI "GET" "/api/v1/bots/{id}" "单个 bot 详情" "" `{"id":1,...}`) }} {{ template "apidoc-row" (mkAPI "PUT" "/api/v1/bots/{id}" "更新 bot；token 变化时重启 listener" `{"name":"prod","description":"...","bot_token":""}` `{"id":1,...}`) }} {{ template "apidoc-row" (mkAPI "DELETE" "/api/v1/bots/{id}" "删除并停止 listener" "" "204") }} {{ template "apidoc-row" (mkAPI "POST" "/api/v1/bots/{id}/enable" "启用 + 启动 listener" "" "200") }} {{ template "apidoc-row" (mkAPI "POST" "/api/v1/bots/{id}/disable" "禁用 + 停止 listener" "" "200") }}

platform 字段取值：

  • telegram：bot_token 形如 123456:ABC-DEF...；自动启动长轮询 listener；支持 inline keyboard、editMessageText、自定义命令、setMyCommands
  • lark（bot_kind=webhook）：bot_token 填完整 Lark 自定义机器人 webhook：https://open.feishu.cn/open-apis/bot/v2/hook/<key>。Channel.chat_id 字段在 Webhook 下无意义；不启动 listener；inline keyboard 等 Telegram 特性静默跳过；editMessageText 自动 fallback 为新发送一条文本消息
  • lark（bot_kind=app）：填 app_id+app_secret+encrypt_key+verify_token。可通过 IM API 主动发送消息到任意 chat_id，并接收 Lark 事件订阅推送（监听用户消息，触发 Starlark 命令）。app_secret 与 encrypt_key AES-GCM 加密入库

runtime 按 bot_kind 自动路由：app → IM API（用 tenant_access_token），webhook → 自定义机器人 URL，telegram → Bot API。

{{/* ===== COMMANDS ===== */}}

Commands

Telegram 用户在私聊发 /<name> [args...] 触发 Starlark 脚本（沙箱执行）。

{{ template "apidoc-row" (mkAPI "GET" "/api/v1/commands" "列命令" "" `{"items":[{"id":1,"name":"echo","code":"def handle(args): return ...","enabled":true}]}`) }} {{ template "apidoc-row" (mkAPI "POST" "/api/v1/commands" "新建" `{"name":"echo","description":"echo args","code":"def handle(args):\n return ' '.join(args)","enabled":true}` `{"id":1,...}`) }} {{ template "apidoc-row" (mkAPI "GET" "/api/v1/commands/{id}" "详情" "" `{"id":1,...}`) }} {{ template "apidoc-row" (mkAPI "PUT" "/api/v1/commands/{id}" "更新" `{"name":"echo","code":"...","enabled":true}` `{"id":1,...}`) }} {{ template "apidoc-row" (mkAPI "DELETE" "/api/v1/commands/{id}" "删除" "" "204") }} {{ template "apidoc-row" (mkAPI "POST" "/api/v1/commands/{id}/test" "沙箱试跑；不写 subscriber" `{"args":["hello"]}` `{"output":"...","return":"hello","error":""}`) }}

Starlark 可用模块：time.now()、json.encode/decode、http.get(url)、http.post(url, body)、print

脚本必须定义 handle(args) 函数；执行超时 10s；输出上限 8KB；代码上限 64KB；HTTP 不允许私网/链路本地/metadata，不跟 redirect。

{{/* ===== LARK EVENTS ===== */}}

Lark 事件订阅

飞书开放平台 → "事件订阅" 配置回调地址；body 签名（HMAC）即身份认证，无需 PulseGuard session。

{{ template "apidoc-row" (mkAPI "POST" "/api/v1/lark/events" "事件回调入口（公网）" `{"type":"url_verification","token":"","challenge":"abc"}` `{"challenge":"abc"}`) }}

配置流程：

  1. 在 PulseGuard 创建 Bot 时选 platform=lark、bot_kind=app，填 app_id / app_secret / encrypt_key / verify_token（与飞书"凭证与基础信息" + "事件订阅"页签一致）
  2. 在飞书"事件订阅"页填回调地址 https://<your-host>/api/v1/lark/events
  3. 点"保存"，飞书会发 url_verification 探测包；PulseGuard 校验 verify_token + 回声 challenge 完成握手
  4. 在"事件订阅 → 添加事件"开启 im.message.receive_v1（接收消息）

签名校验：飞书每个事件请求都带 X-Lark-Signature 头（HMAC-SHA256(encrypt_key, timestamp + nonce + body)）；PulseGuard 用对应 bot 的 encrypt_key 重算并 constant-time 比对，签名不匹配的请求 401 拒绝。

命令调度：收到 im.message.receive_v1 后，按文本首词匹配本租户已注册的 Starlark 命令；命中即在 10s 沙箱里执行，输出回贴到原对话。未命中静默丢弃。

该端点位于公开 /api/v1 路径下，仍受全局 IP 限流保护（默认 60 req/min/IP）。
